# Wokwi diagram.json 排版规范

基于用户实际排版风格总结，后续生成 diagram.json 必须遵循。

---

## 一、元件布局

### 1. 核心板
- ESP32 固定放左上角，top 和 left 接近 (0, 0)，实际用小数如 (4.7, 4.6) 也可以。

### 2. LED
- LED 竖直排成一列，left 值完全相同。
- 从上到下排列，间距均匀。
- 统一加 `flip: "1"`，让引脚朝向合理。

### 3. 电阻
- **电阻横放（rotate: 0）**，不竖放。
- 每个电阻放在对应 LED 的左侧，切近一些。
- 电阻的 top 比 LED 的 top 稍偏下一点，这样电阻右端引脚和 LED 阳极基本在同一水平线上。

### 4. 整体关系
```
ESP32(左上)                    [R3]─🟡 (最上)
                               [R2]─🟢
                               [R1]─🔴 (最下)
```

LED 列在右，电阻在左，GPIO 从 ESP32 引出后向右连接到电阻。

---

## 二、走线

### 1. 信号线（GPIO → 电阻 → LED）
- 电阻右端 → LED 阳极：固定 `h20`，三条完全一致。
- GPIO → 电阻左端：用尽量简洁的路径，允许用 v/h 组合走到电阻位置。

### 2. 地线（GND）
- 采用"主总线 + 分支引出"方式：
  - 先从 GND.1 走一段统一的路径出来
  - 然后分别向各 LED 阴极分支
- 分支路径尽可能统一，保持类似结构。

### 3. 颜色
- 绿色 → 信号线（GPIO）
- 黑色 → 地线（GND）

### 4. 所有走线都是直角，无斜线。

---

## 三、坐标与路径

- 坐标**允许使用小数**，不强制整除 10。
- 路径指令（v/h）**允许使用小数**。
- 重点是**视觉整齐、走线清晰、不交叉**，绝对坐标值灵活即可。
