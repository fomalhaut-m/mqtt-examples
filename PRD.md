# MQTT 多设备模拟系统

## 一、基本信息

- **产品名称**：MQTT 多设备模拟系统
- **技术栈**：Vue3（控制台）+ SpringBoot（服务端）+ H2（设备信息）+ ClickHouse（时序数据）+ EMQX（MQTT Broker）
- **系统架构**：前后端分离，服务端作为 MQTT 客户端
- **核心能力**：多虚拟设备管理、定时上报模拟、指令接收与响应、实时数据监控、数据持久化

**项目结构**：
- Web 控制台：设备控制模块（独立）、数据展示模块（独立）
- SpringBoot 服务：模拟设备管理、MQTT 客户端、MQTT 数据采集（三个独立模块）
- Docker 部署：EMQX Broker（含 docker-compose.yml）

## 四、模块隔离说明

### Web 控制台模块隔离
| 模块 | 职责 | 数据源 |
|-----|------|-------|
| 模拟设备控制模块 | 设备管理、启停控制、上报配置 | 调用 SpringBoot HTTP API |
| 实时数据展示 & 监控模块 | 数据展示、图表、统计、报警 | 调用 SpringBoot HTTP API / SSE |

### SpringBoot 服务端模块隔离
| 模块 | 职责 | 数据存储 |
|-----|------|---------|
| 模拟设备管理模块 | 设备CRUD、定时上报调度、状态管理 | H2 数据库（文件保存） |
| MQTT 客户端模块 | MQTT连接管理、数据上报、指令订阅 | 无持久化 |
| MQTT 数据采集模块 | 订阅上报Topic、解析数据、实时推送 | ClickHouse |

## 五、通信方式

| 通信链路 | 协议 | 端口 | 说明 |
|---------|------|------|------|
| SpringBoot ↔ EMQX | MQTT TCP | 1883 | 设备上报、指令订阅、数据采集 |
| Web ↔ SpringBoot | HTTP | 8081 | 控制指令、数据查询 |
| Web ↔ SpringBoot | SSE | 8081 | 实时数据推送 |
| Web ↔ SpringBoot | WebSocket | 8081 | 设备状态推送 |

## 六、模块划分

### 1. Web 控制台（Vue3）

#### 1.1 模拟设备控制模块（独立）
- 设备列表（ID、名称、类型、状态）
- 设备增删改查
- 启动/停止设备（控制定时上报）
- 设置上报间隔
- 手动触发单次上报
- 设备筛选（在线/离线）

#### 1.2 实时数据展示 & 监控模块（独立）
- 实时数据流（最新接收的数据）
- 设备数据列表
- 图表可视化（折线图、柱状图）
- 数据统计（接收总数、在线设备数、数据速率）
- 报警管理：
  - 手动触发报警
  - 自动报警（指标阈值）
  - 报警开关控制

### 2. SpringBoot 服务端

#### 2.1 模拟设备管理模块（独立）
- 设备信息 CRUD（存储到 H2 数据库）
- 设备状态管理（在线/离线）
- 定时上报调度（内部定时器）

#### 2.2 MQTT 客户端模块（独立）
- 连接 EMQX Broker（TCP 1883）
- 自动重连机制
- 按设备上报数据到指定 Topic
- 订阅设备指令 Topic

#### 2.3 指令接收与处理
- 订阅设备专属指令 Topic：`device/${deviceId}/cmd`
- 解析指令类型并执行：
  - `start_report`：启动自动上报
  - `stop_report`：停止自动上报
  - `set_interval`：修改上报间隔
  - `offline`：设备下线
  - `custom`：自定义透传指令
- 指令回执：向 `device/${deviceId}/reply` 推送执行结果

#### 2.4 MQTT 数据采集模块（独立）
- 订阅设备上报 Topic：`device/${deviceId}/reporter`
- 接收来自模拟设备或真实设备的数据
- 解析并存储到 ClickHouse
- SSE 实时推送给 Web

## 九、交互规则

1. Web 控制台两个模块独立运行，互不干扰
2. SpringBoot 三个模块独立运行，互不干扰
3. 设备信息存储到 H2（文件保存），上报数据存储到 ClickHouse
4. 设备启动时，服务端创建定时器进行定时上报
5. 设备停止时，服务端停止定时器，取消指令订阅
6. 删除设备时，服务端清理定时器和订阅，从 H2 删除记录
7. 编辑设备配置实时生效

## 十、异常处理

| 场景 | 处理方式 |
|-----|---------|
| MQTT 连接断开 | 自动重连，设备状态标记为离线 |
| Web 与服务端断开 | 显示连接状态，尝试重连 |
| 收到非法格式指令 | 日志记录，不回执 |
| 指令 Topic 订阅失败 | 日志告警，上报功能正常 |
| 数据解析失败 | 记录原始数据，标记解析异常 |
| ClickHouse 写入失败 | 内存缓存重试，记录错误日志 |
| H2 写入失败 | 返回错误信息，记录日志 |

## 十一、非功能性需求

- 设备数量支持：至少 100 台
- 单设备上报频率：最高 1 次/秒
- SSE 并发连接：支持 500+ Web 客户端
- 数据延迟：端到端 < 500ms
- ClickHouse 查询响应：< 1s（单设备一天数据）
- WebSocket/SSE 断线重连：自动重连，最多重试 5 次
